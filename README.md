## 這是什麼

本系統是一個**單頁網頁應用程式**，所有功能都在一個 `index.html` 加上一組 `js/` 模組裡。老師只要把整包檔案放上任何能開網頁的地方（甚至直接用瀏覽器開檔），就能擁有一個班級專屬的親師互動網站。

核心特色：

- **零後端**：資料存在 Google Firebase 雲端，或存在瀏覽器本機，兩者皆免費。
- **即時同步**：雲端版老師一更新，家長端畫面自動刷新（採用 Firestore 即時監聽）。
- **行動裝置友善**：手機、平板、電腦皆可使用。
- **模組化權限**：每個功能可獨立設定「公開／需登入／關閉」。
- **一鍵分享**：產生含設定的分享連結與 QR Code，家長掃碼即可連上。

## 兩種使用模式

系統第一次開啟時會請你選擇資料儲存方式，兩種模式可隨時在「設定」中互相切換：

| | ☁️ **Firebase 雲端版** | 💻 **純本地版** |
|---|---|---|
| 資料儲存位置 | Google Firebase 雲端 | 老師自己的瀏覽器（LocalStorage） |
| 家長 / 學生可否查看 | ✅ 可以（多裝置即時同步） | ❌ 不行（只有老師本機看得到） |
| 需要網路 | 需要 | 不需要 |
| 需要申請帳號 | 需要 Firebase（免費） | 不需要 |
| 多班級切換 | 一組設定一個班級 | ✅ 支援多班級、可匯出/匯入備份 |
| 適合對象 | 想讓家長在家查看的班級 | 只想自己上課用、或不想連網的老師 |

> **一句話判斷**：要讓**家長在家連上來看**，就用 **Firebase 雲端版**；只是自己上課當小工具用，就用**本地版**。

## 雲端版：申請 Firebase 並填入網頁（完整圖解步驟）

Firebase 是 Google 提供的免費雲端資料庫服務。以下流程約 10 分鐘可完成，**全程免費、不需綁信用卡**。

### 第 1 步：建立 Firebase 專案

1. 用電腦瀏覽器前往 **[Firebase 主控台](https://console.firebase.google.com)**，使用你的 Google 帳號登入。
2. 點選「**新增專案**」。
3. 輸入專案名稱（例如 `my-class-web`），按「**繼續**」。
4. Google Analytics 可以**關閉**（不需要），然後按「**建立專案**」，等待約 30 秒完成。

### 第 2 步：建立 Firestore 資料庫

1. 進入專案後，從左側選單點 **建構（Build）→ Firestore Database**。
2. 點「**建立資料庫**」。
3. 選擇「**以測試模式啟動（Start in test mode）**」。
4. 地區（Location）建議選 **`asia-east1`（台灣 / 彰化）**，按「**啟用**」。

### 第 3 步：啟用匿名登入

> 系統需要「匿名登入」才能讓家長端在不註冊的情況下讀寫資料。

1. 從左側選單點 **建構（Build）→ Authentication**，按「**開始使用**」。
2. 切到「**Sign-in method**」分頁。
3. 在清單中找到「**匿名（Anonymous）**」，點開並把開關**啟用**，儲存。

### 第 4 步：取得網頁設定（firebaseConfig）

1. 回到左上「**專案總覽**」。
2. 在頁面中央找到 **`</>`（網頁）** 圖示，點它新增一個網頁應用程式。
3. 輸入應用程式暱稱（隨意），按「**註冊應用程式**」。
4. 畫面會出現一段程式碼，找到 `const firebaseConfig = { ... }` 這個物件，內容類似：

```js
const firebaseConfig = {
  apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXX",
  authDomain: "my-class-web.firebaseapp.com",
  projectId: "my-class-web",
  storageBucket: "my-class-web.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcd1234efgh5678"
};
```

### 第 5 步：把設定貼進網頁

1. 開啟本系統的 `index.html`。第一次開啟時選「**Firebase 雲端**」，或在已開啟的頁面右上角點「**⚙️ 設定**」。
2. 在「**貼上 Firebase 設定**」的方框中，貼上**大括號 `{ }` 之間的六行內容**：

```
apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXX",
authDomain: "my-class-web.firebaseapp.com",
projectId: "my-class-web",
storageBucket: "my-class-web.appspot.com",
messagingSenderId: "1234567890",
appId: "1:1234567890:web:abcd1234efgh5678"
```

> 💡 貼**整段含大括號**也可以，系統會自動解析。最低限度只要有 `apiKey` 與 `projectId` 兩行即可連線。

3. 按「**解析並儲存（重新整理）**」。頁面會自動重整並連上雲端，右上角連線狀態顯示已連線即代表成功。

### 第 6 步（建議）：設定安全規則

測試模式的規則會在 30 天後過期。正式使用前，請到 **Firestore Database → 規則（Rules）** 貼上以下內容並「**發布**」：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.time < timestamp.date(2126, 7, 24);
    }
  }
}
```

> ⚠️ **安全性說明**：座號密碼與老師密碼屬於「前端輕量保護」，適合一般班級的非機敏用途（聯絡簿、作業、加扣分等）。請勿存放高度隱私或機敏資料。若需更高安全性，請改用 Firebase 自訂後端驗證。

### 第 7 步：分享給家長

1. 在「**⚙️ 設定**」視窗中點「**產生分享連結**」。
2. 系統會產生一條含有你 Firebase 設定的網址、短網址，以及一個 **QR Code**。
3. 把連結或 QR Code 發給家長，他們用手機開啟後就會自動連上同一個班級資料庫，**無須自行設定 Firebase**。

> 家長端打開後，可用「座號 + 專屬密碼」登入查看自己孩子的資料（密碼由老師在後台建立學生名單時設定）。

## 本地版：完整使用說明（無家長功能）

本地版把所有資料存在**老師這台電腦的瀏覽器**裡，完全免連網、免申請、免費。

> ⚠️ **重要前提**：本地版的資料只存在這台瀏覽器，**家長與學生無法連上查看**。家長登入、私訊、回條回傳等「需要家長端」的功能在本地版不會運作（登入視窗也只保留老師登入）。本地版定位為**老師個人的課堂工具與班務管理**。

### 啟用本地版

1. 第一次開啟 `index.html` 時，在歡迎畫面選「**💻 純本地使用**」。
2. 輸入班級名稱，建立第一個班級即可開始使用。

### 多班級管理

本地版支援同時管理多個班級（例如帶兩個班、或不同年度）。

- 進入「**⚙️ 設定**」→「**📚 班級管理**」。
- 可**新增**班級、**切換**目前使用的班級、**改名**、**刪除**。
- 每個班級的資料各自獨立，互不干擾。

### 資料備份與還原（務必定期做）

> ⚠️ 瀏覽器的本機儲存可能在**清除快取、更換瀏覽器、重灌電腦**時消失。請**定期匯出備份**以免資料遺失。

在「**⚙️ 設定**」→「**💾 資料備份與還原**」中：

- **匯出目前班級**：把目前班級的所有資料存成一個 `.json` 檔下載。
- **匯出全部班級**：一次備份所有班級。
- **匯入備份檔案**：選取先前匯出的 `.json` 檔還原資料（可跨電腦搬移）。

### 老師登入（本地版）

- 點右上角「**登入**」，輸入老師密碼。
- **首次登入**：輸入任意密碼即會自動建立為該班級的老師後台密碼。
- 登入後即可使用後台與所有教學工具。

### 切換到雲端版

若日後想開放給家長查看，可在「**⚙️ 設定**」中點「**☁️ 切換為 Firebase 雲端模式**」。本地班級資料不會被刪除，可先匯出備份，未來切回本地版再匯入。

## 登入與權限說明

系統有三種身分：

| 身分 | 登入方式 | 可做什麼 |
|---|---|---|
| **訪客** | 不需登入 | 查看設定為「公開」的模組 |
| **家長 / 學生** | 座號 + 專屬密碼（老師設定）<br>*僅雲端版* | 查看需登入的模組、看自己孩子的成績/點數、回傳回條、私訊老師 |
| **老師** | 老師後台密碼 | 完整管理權限、後台、所有教學工具 |

每個模組可由老師在後台獨立設定三種權限：

- **公開（public）**：所有人（含未登入訪客）皆可查看。
- **需登入（login）**：須家長/學生或老師登入後才可查看。
- **關閉（off）**：完全隱藏該模組。

老師亦可在後台調整各模組在主畫面的**顯示順序**。

## 部署到 GitHub Pages

想把這個網站放上網路、產生一個公開網址，最簡單的方式是用免費的 **GitHub Pages**：

1. 在 GitHub 建立一個新的 **repository**（例如 `class-web`），設為 **Public**。
2. 把本專案所有檔案（`index.html`、`js/` 資料夾、`README.md` 等）上傳到 repository。**請保持資料夾結構不變**（`index.html` 與 `js/` 在同一層）。
3. 進入 repository 的 **Settings → Pages**。
4. 在「Build and deployment」的 Source 選「**Deploy from a branch**」，Branch 選 `main`、資料夾選 `/ (root)`，按 **Save**。
5. 等待約 1 分鐘，頁面上方會出現你的網址，格式為 `https://你的帳號.github.io/class-web/`。
6. 用這個網址開啟，依上方步驟設定 Firebase 即可。

> 💡 用 GitHub Pages（`https://`）開啟，可避免直接用 `file://` 開檔時的瀏覽器限制（例如圖床上傳、部分同步功能）。

### 本機測試（選用）

若想在自己電腦先測試，建議用簡易伺服器開啟，避免 `file://` 限制：

```bash
# 在專案資料夾中執行
python -m http.server 8000
# 然後瀏覽器開 http://localhost:8000
```

### 技術說明

- 所有 JS 為傳統 `<script>` 載入、共用同一全域範圍，無需打包工具。
- 載入順序已在 `index.html` 底部排定（本地資料層 → 基礎設定 → 核心 → 各模組）。
- 外部相依（皆由 CDN 載入）：Tailwind CSS、Noto Sans TC 字體、Firebase v8 SDK、JSZip、QRCode；PDF 相關（jsPDF / html2canvas）於需要時動態載入。
- 更新程式後，記得調整 `index.html` 中 `<script src="...?v=">` 的版本參數，避免瀏覽器讀到舊快取。

## 常見問題 FAQ

**Q：一定要用 Firebase 嗎？會收費嗎？**
不一定。只想自己上課用就選本地版。Firebase 免費額度對一般班級綽綽有餘，**不需綁信用卡**。

**Q：家長需要自己申請 Firebase 嗎？**
不用。老師設定好後產生「分享連結 / QR Code」給家長，家長開啟即自動連上同一個班級。

**Q：家長要怎麼登入看自己孩子的資料？**
用「座號 + 專屬密碼」登入，密碼由老師在後台建立學生名單時設定。

**Q：本地版可以給家長看嗎？**
不行。本地版資料只存在老師這台瀏覽器。要開放家長請改用雲端版。

**Q：資料會不會不見？**
雲端版存在 Google 雲端，相對安全。本地版存在瀏覽器，**請務必定期匯出備份**。

**Q：忘記老師密碼怎麼辦？**
