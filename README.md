## 🎯 為什麼需要這個工具？

身為導師，您是否也有這些困擾？

- **LINE 群組訊息洗版** — 重要公告被聊天訊息淹沒，家長根本沒看到
- **聯絡簿常忘記簽** — 紙本聯絡簿容易遺失，也無法確認家長是否已讀
- **作業繳交狀況混亂** — 每天要人工登記誰沒交作業，耗時又容易出錯
- **回條回收困難** — 紙本回条常遺失，還要手動統計誰交誰沒交
- **成績通知不便** — 小考成績想讓家長知道，但又不想在群組公開個資
- **課堂管理工具分散** — 抽人、計時、加分扣分，要用好幾個不同工具

**班級親師互動網** 將所有這些需求整合在一個網頁中，老師不必學習多套系統，家長也不用安裝任何 App，打開瀏覽器就能用。

## 💌 雲端版：可取代 LINE 的親師溝通方案

### 為什麼不用 LINE 群組？

| 項目 | LINE 群組 | 本系統雲端版 |
|---|---|---|
| 訊息分類 | 全部混在一起，公告被聊天洗掉 | 聯絡簿、公告、成績各自獨立區塊 |
| 個資保護 | 成績、回條在群組公開 | 家長僅見自己孩子的資料 |
| 已讀確認 | 無法區分誰已讀 | 聯絡簿有「我已閱讀」按鈕，教師可查看已讀清單 |
| 私訊老師 | 需加好友，老師不堪其擾 | 內建一對一私訊，老師在後台統一管理 |
| 作業繳交 | 無法追蹤 | 即時檢核表格，一目瞭然 |
| 回條回收 | 需一一核對 | 線上拍照上傳，教師即時審核 |
| 檔案整理 | 照片訊息會過期 | 可打包 ZIP／PDF 下載所有回條 |

### 家長私訊 + Email 通知流程

這是本系統取代 LINE 的關鍵功能：

```
家長傳送訊息
    │
    ├──→ 訊息存入 Firebase（老師登入後台即可查看）
    │
    └──→ Web3Forms 自動寄送 Email 通知老師
              │
              └──→ 老師 Email 收到通知信
                     主旨：【三年一班】5號 王小明的家長 傳來私訊
                     內容：家長姓名、座號、訊息內容
```

**老師不需要加家長 LINE 好友**，家長也不需要加老師好友，  
所有溝通都在系統內完成，**對話紀錄永久保存**，不怕換手機遺失。

### 設定 Email 通知只需 3 步

1. 前往 [Web3Forms](https://web3forms.com)，輸入您的 Email，取得免費 Access Key
2. 在老師後台 → 設定 → 貼上 Access Key，點「✉️ 測試」確認收信正常
3. 勾選「家長傳送私訊時自動寄通知信」→ 儲存設定

完成後，家長只要傳送私訊，系統就會在 **10 分鐘內最多寄送一封通知信**（避免洗版），  
確保老師不會錯過任何家長的訊息。

## 📊 兩種模式比較

| 比較項目 | ☁️ 雲端版（Firebase） | 🖥️ 純本地版 |
|---|---|---|
| 是否需要網路 | ✅ 需要（老師與家長皆需連線） | ❌ 完全離線可用 |
| 資料儲存 | Firebase 雲端資料庫 | 瀏覽器 localStorage |
| 多裝置同步 | ✅ 即時同步（手機、平板、電腦） | ❌ 限單一裝置 |
| 家長遠端觀看 | ✅ 家長可在家用手機連線 | ❌ 限同一臺電腦 |
| 設定難度 | 需申請 Firebase 免費帳號（約 10 分鐘） | 免設定，開啟即用 |
| 多班級支援 | ✅ 老師可管理多班 | ✅ 可建立多個獨立班級 |
| 資料備份 | Firebase 自動雲端備份 | 手動匯出 JSON 備份 |
| 適用場景 | 導師長期經營、家長遠端參與 | 科任老師、教室無網路、課堂即時操作 |

> 兩個版本的操作介面完全相同！可先用本地版熟悉，再無痛轉移至雲端版。

│   ├── 80-tools.js              模組：課堂小工具

**技術棧：**
- 前端框架：Tailwind CSS（CDN）
- 資料庫：Firebase Firestore v8（Cloud）／ localStorage（Local）
- 認證：Firebase Auth 匿名登入
- 圖床：imgbb / freeimage.host / SM.MS
- 郵件通知：Web3Forms API
- PDF 輸出：jsPDF + html2canvas（動態載入）
- 壓縮下載：JSZip

## ⚡ 快速開始

### 本地版（30 秒上手）

1. 下載本專案資料夾，保持 `index.html` 與 `js/` 的相對位置
2. 用 Chrome 或 Edge 雙擊 `index.html` 即可開啟
3. 點「⚙️ 系統設定」→ 選擇「純本地模式」→ 輸入班級名稱
4. 開始使用！

> 建議使用簡易 HTTP 伺服器以啟用完整功能：
> ```
> python -m http.server 8000
> ```
> 然後在瀏覽器開啟 `http://localhost:8000`

### 雲端版（需申請 Firebase）

1. 前往 [Firebase 控制台](https://console.firebase.google.com) 新增專案
2. 建立 Firestore Database（測試模式）並啟用 Authentication 匿名登入
3. 新增 Web 應用程式，複製 `firebaseConfig` 金鑰
4. 開啟 `index.html` → 點「⚙️ 設定」→ 貼入金鑰 → 連線完成
5. 將產生的分享連結或 QR Code 發給家長

## 📖 給老師的完整功能說明

### 每日聯絡簿
- 支援 Markdown 語法（**粗體**、清單、超連結）
- 行首「`- `」的作業項目會自動同步到作業檢核模組
- 家長點「✋ 我已閱讀」後，教師可查看已讀清單
- 支援全螢幕放大顯示（方便學生抄寫）

### 作業檢核
- 自動從聯絡簿帶入作業項目，不必重複輸入
- 四色狀態：灰色未繳交、綠色已繳交、黃色待訂正、藍色已訂正
- 可點欄標題批次設定全班狀態，或拖曳選取連續區塊

### 紙本回條回傳
- 家長拍照上傳，支援自動壓縮
- 教師即時檢視全班縮圖牆（顏色標示狀態）
- 可逐張審核（核准／退回），或批次下載 ZIP／PDF

### 班級日曆
- 以月曆形式呈現，可切換月份
- 點選日期可新增或查看活動
- 適合標註段考、運動會、校外教學等重要日期

### 課堂加扣分
- 後台自訂行為項目（如：回答問題 +1、作業未交 -1）
- 前台點擊學生卡片即可加扣分，適合課堂即時操作
- 全班排行以座號匿名顯示（僅自己孩子顯示姓名）
- 小組加扣分模式：建立小組後以組為單位加扣分
- 全螢幕卡牆模式：大尺寸卡片適合投影展示

### 班級座位表
- 兩種模式：按列排列（固定格線拖曳互換）／ 自由排列（任意拖曳）
- 可匯出座位表 PDF，方便張貼於教室

### 課堂小工具
- **隨機抽人**：從全班隨機抽選，支援不重複模式
- **倒數計時器**：快速設定常見時間（1/2/3/5/10 分鐘），全螢幕大字顯示

### 老師後台
13 個管理分頁，完整掌控所有資料：
設定 → 學生帳號 → 聯絡簿 → 作業檢核 → 公告 → 日曆 → 回條 → 私訊 → 資源連結 → 小考成績 → 加扣分 → 座位表 → 小工具
