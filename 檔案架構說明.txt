【說明】
原本整個程式（HTML 結構、CSS 樣式、所有 JavaScript）都寫在單一個
index.html 檔裡。本「功能分割版」把當中的 JavaScript 依功能拆成多個
獨立的 .js 檔，主檔 index.html 只保留 HTML 結構與 CSS 樣式，並依正確
順序用 <script src="..."> 逐一載入這些 JS 檔。

操作方式與單檔版相同。可直接用瀏覽器開啟「功能分割版」資料夾裡的
index.html 使用（請保持 index.html 與 js 資料夾的相對位置不變）。

本系統支援兩種資料來源：
  ‧ Firebase（雲端）模式：多裝置即時同步。
  ‧ 純本地模式：以瀏覽器 localStorage 儲存，免連線、可多班級切換
    （由 00-local-db.js 提供與 Firestore 相容的 API shim）。

│ 各檔案功能說明                                                         │
└──────────────────────────────────────────────────────────────────────┘

■ index.html　（主畫面）
  ‧ 網頁的 HTML 結構：頂部導覽列、各功能模組區塊、頁尾、浮動按鈕、
    模態視窗與 Toast 容器。
  ‧ 全部 CSS 樣式（淡色卡片、3D 按鈕、聯絡簿、作業表格、加扣分／座位卡片等）。
  ‧ 外部資源：Tailwind、Noto Sans TC 字體、Firebase v8 SDK、JSZip、QRCode；
    PDF 相關（jsPDF / html2canvas）於需要時動態載入。
  ‧ 最後依序用 <script src="js/..."> 載入下方所有 JS 檔，並以 ?v= 版本參數
    避免瀏覽器載入舊的快取（更新程式後請記得調整版本參數）。

■ js/00-local-db.js　（純本地模式資料庫層）
  ‧ 提供與 Firebase Firestore (v8 compat) 相容的 API shim，讓其餘模組
    完全不需改動即可在不連線 Firebase 時，以 localStorage 儲存所有資料。
  ‧ 支援多班級：每個班級資料以不同 prefix 隔離，含建立／刪除／改名／
    切換班級、以及整班資料的匯出／匯入（window.LocalDB）。

■ js/01-config-core.js　（設定與全域狀態）
  ‧ Firebase 設定區、模式常數（雲端／本地）、LocalStorage 鍵名常數。
  ‧ Toast 通知、通用模態視窗 showModal/closeModal、確認對話框 confirmDialog。
  ‧ 設定來源解析（網址參數 > 寫死 > LocalStorage）。
  ‧ 全域狀態 APP_STATE、各模組共用變數、權限與排序預設值
    （DEFAULT_PERMS / MODULE_NAMES / DEFAULT_ORDER / ORDER_LABELS 等）。
  ‧ ★此檔須最先載入，其餘各檔的函式皆共用此處宣告的全域變數。

■ js/02-app-core.js　（應用核心）
  ‧ 啟動流程 init（DOMContentLoaded 進入點）、依模式連線 Firebase 或本地。
  ‧ 主程式 startApp：建立各資料即時監聽、呼叫各模組 setup。
  ‧ 權限判斷 permOf / canView / isLoggedIn。
  ‧ 模組顯示與排序、整體刷新 refreshActivePanels。

■ js/03-auth.js　（登入／登出）
  ‧ 登入視窗（家長／學生分頁、老師分頁）。
  ‧ 家長登入、老師登入、登出、登入狀態還原與登入列 UI。

■ js/04-utils.js　（共用工具）
  ‧ 日期字串、HTML 跳脫 escapeHtml、富文本 renderRich、圖片壓縮 compressImage。

■ js/10-contactbook.js　（每日聯絡簿）
  ‧ 前台聯絡簿顯示、放大全螢幕抄寫模式（含字級調整）、家長「我已閱讀」、
    教師檢視家長已讀狀況。

■ js/11-homework.js　（作業檢核）
  ‧ 今日作業繳交即時狀況表格、教師主畫面「立即檢核」模式、
    從聯絡簿自動帶出作業項目。

■ js/12-calendar.js　（班級日曆）
  ‧ 前台月曆呈現、上下月切換、活動點選查看。

■ js/13-announcements.js　（公告與榮譽榜）
  ‧ 前台公告清單顯示（可展開／收合）、點圖放大檢視。
  ‧ 每則公告的標題採向左對齊呈現。

■ js/14-slips.js　（回條拍照回傳）
  ‧ 前台回條清單、家長拍照上傳回條（含壓縮）、教師端內嵌縮圖審核。
  ‧ 教師下載：
      - 下載全部圖片（ZIP，downloadSlipImagesZip）。
      - 下載 PDF（downloadSlipImagesPdf）：把所有回傳圖片轉成一張一頁的
        PDF，每頁上方標示上傳者姓名（座號＋姓名），jsPDF 動態載入。

■ js/15-resources.js　（資源充電站）
  ‧ 前台資源連結卡呈現，以及後台新增／編輯／刪除連結。

■ js/16-quiz.js　（今日小考成績）
  ‧ 前台依日期查看成績（家長僅見自己孩子）、後台批次匯入／單筆輸入、
    AI 整理提示詞、成績發布與刪除。

■ js/20-visitor-help-settings.js　（訪客／說明／設定）
  ‧ 不重複訪客統計、使用說明視窗。
  ‧ Firebase 設定／分享視窗（產生分享連結與 QR Code、複製等）。
  ‧ 純本地模式設定視窗（班級管理、資料匯出匯入）。

■ js/30-admin.js　（老師後台）
  ‧ 後台總框架 openAdmin / renderAdminTab（分頁切換）。
  ‧ 班級設定與權限矩陣、模組顯示順序、後台密碼、Email 通知設定。
  ‧ 學生帳號管理（新增／編輯／刪除／批次匯入）。
  ‧ 各模組後台：聯絡簿、作業檢核、公告、日曆、回條的管理介面。
  ‧ 自動刪除設定：
      - 每則公告／每筆回條可「各自獨立」設定到期日（expireAt，
        建立時填或事後以 ⏱️ 按鈕修改），到期後於老師開啟後台時自動
        刪除（含圖片；回條會一併刪除繳交資料）。
      - 另保留「全域預設天數」設定（公告 / 回條各一），作為未指定單則
        到期日時的預設清除規則。
      - 清理函式 _runAutoDel / runAllAutoDel：優先依單則 expireAt，
        否則套用全域天數。
  ‧ ★加扣分後台分頁內容函式 adminPoints 定義在 60-points.js；座位表後台
    adminSeating 定義在 70-seating.js，由本檔的 renderAdminTab 呼叫。

■ js/40-quick.js　（主畫面快速新增）
  ‧ 教師不進後台、直接在主畫面快速新增：聯絡簿、作業檢核、活動、公告、
    資源、小考成績、回條。

■ js/50-messages.js　（家長↔老師私訊）
  ‧ 家長端浮動按鈕與對話視窗、傳送訊息、未讀數。
  ‧ 老師端浮動收件匣、依座號分組、開啟對話並回覆、收回訊息、
    匯出私訊紀錄為 HTML。
  ‧ 家長傳訊時自動寄 Email 通知老師（Web3Forms / 免後端）。

■ js/60-points.js　（課堂加扣分）
  ‧ 教師端：學生卡片牆加扣分、自訂分數、查看個人歷史、全螢幕卡牆、
    小組加扣分（小組以圖卡呈現並列出組員）。
  ‧ 家長／學生端：唯讀全班點數排行，僅自己孩子顯示姓名，其餘以座號匿名。
  ‧ 後台「加扣分」分頁 adminPoints：行為項目管理＋學生分組設定＋全班重設。
  ‧ 本檔為獨立 IIFE 封裝，只透過 window.* 對外掛載必要函式；
    啟動時自動把「加扣分」註冊進權限矩陣與模組排序。

■ js/70-seating.js　（座位表）
  ‧ 教師端兩種模式：① 按列排座位（固定格子、拖曳互換）；
    ② 自由排列（拖曳姓名卡到教室任意位置）。
  ‧ 下載座位表 PDF（html2canvas + jsPDF，動態載入）。
  ‧ 後台 adminSeating：設定模式、列／欄數、自動排入、清空座位表。
  ‧ 獨立 IIFE 封裝，透過 window.* 對外掛載。

■ js/80-tools.js　（課堂小工具）
  ‧ ① 隨機抽人（可設「不重複」模式，已抽過者標記排除）。
  ‧ ② 倒數計時器（可全螢幕、超大數字；倒數最後 10 秒有提示音）。
  ‧ 獨立 IIFE 封裝。

■ js/90-imghost.js　（免費圖床上傳）
  ‧ 支援 imgbb / freeimage.host / SM.MS 上傳，回傳圖片網址。
  ‧ 對外 API：ImgHost.showPicker(file, onSuccess)。
  ‧ ※ Catbox / 0x0.st 因 CORS 限制已移除；如需使用請部署到 https:// 伺服器。


┌──────────────────────────────────────────────────────────────────────┐
│ 載入順序與相依說明（重要）                                             │
└──────────────────────────────────────────────────────────────────────┘

‧ 所有 JS 檔都是「傳統 <script>」載入，彼此共用同一個全域範圍，因此
  各檔的函式可以互相呼叫，不需 import/export。

‧ 載入順序已在 index.html 底部排定，原則為：
    本地資料層(00) → 基礎設定/狀態(01) → 圖床(90) → 應用核心(02)
    → 工具/登入(03,04) → 各功能模組(10~16) → 訪客/說明/設定(20)
    → 後台(30) → 快速新增(40) → 私訊(50) → 加扣分(60)
    → 座位表(70) → 小工具(80)

‧ 00-local-db.js 與 01-config-core.js 要在最前面載入
  （00 提供本地資料庫 shim；01 宣告所有共用的全域變數與常數）。
  其餘檔案的程式碼都在「函式內」才會用到這些變數，而函式要等到使用者
  操作或頁面載入完成後才執行，所以中間各檔的先後不影響功能。

‧ 60-points / 70-seating / 80-tools 為獨立 IIFE，啟動時自行向權限矩陣與
  模組排序註冊，並以 window.* 掛載後台與前台所需函式。

‧ 02-app-core.js 會註冊 DOMContentLoaded 事件，待 HTML 載入完成後啟動程式。


┌──────────────────────────────────────────────────────────────────────┐
│ 使用方式                                                               │
└──────────────────────────────────────────────────────────────────────┘

1. 保持資料夾結構：index.html 與 js/ 資料夾放在一起（相對路徑不可變動）。
2. 第一次使用：用瀏覽器開啟 index.html，依畫面指示完成 Firebase 設定，
   或選擇「純本地模式」直接在本機使用。
3. 後續操作（登入、聯絡簿、加扣分、座位表、後台管理…）與單檔版相同。
4. 部分功能（圖床上傳、Firebase 同步）建議以 http(s) 伺服器開啟，
   避免 file:// 的瀏覽器限制；本機可用 python -m http.server 後再開啟。
